Sistema de Rubricas de Avaliação

Nome da Rubrica: Desenvolvimento de Projeto de Software
Critérios Excelente (4) Bom (3) Satisfatório (2) Insuficiente (1) Artefatos Atingidos
Documentação Documentação completa, clara e bem estruturada com todos os requisitos Documentação presente com a maioria dos requisitos Documentação básica com alguns elementos faltando Documentação incompleta ou ausente README.md, Diagramas UML, Especificações
Qualidade do Código Código limpo, bem organizado, seguindo padrões e boas práticas Código organizado com algumas inconsistências menores Código funcional mas com problemas de organização Código desorganizado e difícil de manter Arquivos fonte, Estrutura de pastas, Comentários
Funcionalidades Todas as funcionalidades implementadas e funcionando perfeitamente Maioria das funcionalidades implementadas corretamente Funcionalidades básicas implementadas com limitações Funcionalidades essenciais não implementadas Módulos funcionais, Testes de integração
Testes Cobertura de testes completa com testes unitários e de integração Boa cobertura de testes com algumas áreas não testadas Testes básicos presentes mas cobertura limitada Testes ausentes ou insuficientes Arquivos de teste, Relatórios de cobertura
Interface do Usuário Interface intuitiva, responsiva e acessível Interface funcional e agradável com pequenos ajustes necessários Interface básica funcional mas com problemas de usabilidade Interface confusa ou não funcional Telas, Componentes UI, Design responsivo
Nome da Rubrica: Trabalho em Equipe e Colaboração
Critérios Excelente (4) Bom (3) Satisfatório (2) Insuficiente (1) Artefatos Atingidos
Comunicação Comunicação constante, clara e efetiva entre todos os membros Boa comunicação com alguns momentos de falha Comunicação básica mas irregular Comunicação deficiente ou ausente Atas de reunião, Mensagens, E-mails
Controle de Versão Uso exemplar de Git com commits descritivos e branches organizados Bom uso de Git com algumas inconsistências Uso básico de Git com problemas de organização Uso inadequado ou ausente de controle de versão Histórico de commits, Pull requests, Branches
Divisão de Tarefas Tarefas bem distribuídas e balanceadas entre todos Boa distribuição com pequenos desbalanceamentos Distribuição desigual mas todas as tarefas foram feitas Distribuição muito desigual ou desorganizada Board de tarefas, Issues, Sprints
Resolução de Conflitos Conflitos resolvidos de forma construtiva e profissional Conflitos geralmente bem resolvidos Conflitos resolvidos mas com dificuldades Conflitos não resolvidos ou mal gerenciados Documentação de decisões, Retrospectivas
Participação Participação ativa e consistente de todos os membros Boa participação geral com algumas variações Participação irregular de alguns membros Participação muito desigual ou ausente Logs de atividade, Contribuições no repositório
